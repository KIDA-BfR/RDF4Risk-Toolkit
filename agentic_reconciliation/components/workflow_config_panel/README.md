# RDF4Risk web app notes

React/TypeScript frontend bundle for the RDF4Risk Toolkit pages.

The frontend source now lives at the repository root in `frontend/` so all UI code is grouped outside the agent reconciliation package.

## Running the app

From the repository root:

```bash
npm run start
```

This starts:

- a Python JSON backend at `http://127.0.0.1:8765`
- the Vite frontend

You can start the main app workflow directly with the npm command above.

## Source organization

The frontend source lives in `/frontend/src` and is organized by feature app:

```text
frontend/src/
  index.tsx                         # React entry point + shared theme
  standalone/                       # App shell, navigation, sidebar, backend API bridge
  apps/
    home/                           # Home dashboard app
    matching-table-generator/       # Matching table workflow
    semi-automatic-reconciliation/  # Semi-automatic reconciliation workflow
    agent-reconciliation/           # Agent-based reconciliation workflow
    rdf-generator/                  # RDF generator workflow
    rdf-to-table/                   # RDF to table workflow
  components/
    run/                            # Reusable run/progress components
  assets/                           # Images, animations, and declarations
```

`/frontend/build` is generated by Vite and should remain the stable production output path for the browser frontend.

## Current maintainability notes

- Feature apps are intentionally easy to find under `src/apps/<feature>/` instead of being mixed directly in `src/`.
- Shared or reusable UI pieces should go under `src/components/<domain>/`.
- App-specific helper types/components can stay inside their feature file until they become shared or too large to navigate.
